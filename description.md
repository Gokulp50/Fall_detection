# Project Description: Edge-AI Emergency Healthcare Platform

## 1. Project Overview
The **Edge-AI Emergency Healthcare Platform** is an integrated system designed for elderly fall prediction, real-time physiological verification, and automated emergency coordination. Unlike traditional systems that merely notify a caregiver after detecting a fall, this platform bridges the gap between detection and response. It calculates the severity of an event, coordinates ambulance dispatch, tracks the response progress, and pre-notifies the receiving hospital—all within a unified, event-driven architecture.

## 2. Hardware Components
The wearable node is designed to be a low-power, compact prototype providing continuous monitoring and edge inference. 

- **ESP32 Microcontroller:** Serves as the central processing unit for the wearable. It handles local data sampling, preprocessing, running quantized TinyML inference, sleep scheduling, and Bluetooth/Wi-Fi communication.
- **MPU6050 Accelerometer:** Measures triaxial acceleration to capture impacts, free-falls, posture transitions, and periods of inactivity.
- **MPU6050 Gyroscope:** Measures triaxial angular velocity to characterize rotation and loss of balance.
- **Heart-Rate Sensor:** Continuously or periodically monitors BPM to establish a pre-event baseline, detect post-fall physiological deviations, and analyze recovery states.
- **Battery System:** A rechargeable lithium cell (targeting 500–1000 mAh) optimized through duty-cycled operation.
- **Gateway Device:** A paired smartphone or Wi-Fi router used to synchronize data to the cloud and provide location services without requiring power-hungry GPS modules on the wearable.

## 3. Software Architecture
The software stack spans from embedded machine learning on the edge to a full-stack Next.js web application in the cloud.

### 3.1 Edge AI & Embedded Software
- **TinyML Inference:** A quantized 1D Convolutional Neural Network (CNN) runs directly on the ESP32. It estimates the probability of a fall based on short inertial data windows. Processing locally reduces latency, bandwidth usage, and privacy exposure.
- **Hybrid Verification Engine:** Combines the probabilistic TinyML outputs with deterministic physical thresholds (e.g., checking acceleration peaks and angular velocity). 

### 3.2 Cloud Dashboard (Next.js)
- **EdgeCare Response Platform:** A web-based dashboard used by caregivers, emergency dispatchers, and hospitals. It displays patient profiles, event timelines, real-time severity scores, and live ambulance tracking.

## 4. Implementation Details & Working Mechanism
The system workflow connects continuous monitoring to verified emergency routing, broken down into several stages:

### Step 1: Wearable Sensing & Local Prediction
The MPU6050 and heart rate sensors sample data continuously. When a potential fall is detected, the 100 Hz high-resolution window is fed into the TinyML model. If the probability of a fall exceeds the configured threshold, a "candidate event" is declared.

### Step 2: Patient Status Verification
Inertial evidence alone is not enough to determine clinical urgency. A post-fall verification process runs:
1. **Impact Check:** Validates the physical impact against hard thresholds.
2. **Physiological Check:** Compares post-event heart rate against the user's personalized rolling baseline to detect abnormal recovery, stress, or syncope.
3. **Inactivity Check:** Determines if the user is motionless.

### Step 3: Emergency Severity Assessment (ESS)
A calculated Emergency Severity Score (ESS) is generated by combining:
- Model confidence
- Peak acceleration and gyroscope readings
- Heart-rate deviation
- Prolonged inactivity
- User response/cancellation status 

The event is classified into one of three levels: **Mild, Moderate, or Critical**.

### Step 4: Coordinated Emergency Response
- **Mild:** The event is logged and sent as a notification to caregivers. Low severity avoids false alarm fatigue.
- **Moderate / Critical:** The cloud server confirms the patient's location and checks caregiver response. It then requests an ambulance dispatch. 

### Step 5: Live Tracking and Hospital Pre-Notification
Once an ambulance is assigned:
- Authorized users receive real-time updates (e.g., en route, arrived at patient, transporting).
- The receiving hospital is automatically pre-notified. The system forwards the patient profile, event time, severity score, heart-rate trend, and Estimated Time of Arrival (ETA), allowing the medical team to prepare before the patient even arrives.

## 5. Performance and Expected Outcomes
By combining threshold-based checks with TinyML and physiological data, the platform aims to:
- Achieve **≥95% accuracy, recall, and specificity**.
- Drastically **reduce false alarms** common in basic fall-detection devices.
- Deliver alerts to caregivers in **under 10 seconds** and prep critical dispatch in **under 30 seconds**.
- Maintain a **24–72 hour battery life** through intelligent sleep scheduling and adaptive sampling.
